# Summary and reflection
In this session, we learned how to make our package executable through a command-line
interface and how to generate professional documentation. Honestly, I’m surprised by how far
we’ve made it! Before this class, even though I had learned some basic Python, I never knew 
how to actually "package" my code to publish or share it.

Every time I read a research paper, used someone else's package to process my data, or 
scrolled through GitHub, I always wondered how they developed such useful toolkits. I used to 
look at how they organized their files into a package that anyone could download and feel a 
bit lost. I didn't fully understand what I was doing when I simply followed their installation 
instructions. But right now, I actually know how to do it myself!

Now, the next time I see an author asking for help updating a package, or if I find an error and 
want to provide feedback, I’ll actually know the process behind it. During this session, I also 
learned that there is a specific career called a Research Software Engineer. I think it would be 
so interesting to develop packages like this for the community. Even if I’m not yet sure if I can 
contribute a "meaningful" or "helpful" message to the world of software, it is still so 
interesting to play around and build something functional.
